Adding New Locations

Description

This use case focuses on allowing users to add new locations to the system. Whether for navigation, place bookmarking, or future route planning, users can introduce custom destinations to improve their travel experience and expand the system's location database.

Key Features

Location Input: Users can input the name and edges of the node [distances].

Location Validation: The system validates new locations to ensure they are accurate and unique.

Data Structures Used

Adjacency List: Locations are stored as nodes within a graph, allowing easy traversal and pathfinding.

fstream[file Handling]: Locations are stored in a local file with details like name, distances.

Indexing: Locations are indexed by their names and categories for efficient searching.

Benefits

Enables users to personalize their travel experience by adding locations of interest.

Improves the system's versatility by expanding its location database.

Enhances route optimization and planning by incorporating user-added destinations.
