Algorithmic Concepts

  • Decomposition: Breaks a large problem into smaller sub-problems, making it easier to manage and solve. Applied in zombie child detection (Lab 1).
  • Pattern Recognition: Identifies recurring patterns to generalize solutions. Demonstrated in representation system activity and Forts of India portfolio.
  • Abstraction: Simplifies problems by focusing on key features. Used in shapes and sizes activity and Guernica abstract model.
  • Brave and Cautious Travel: Graph traversal methods - DFS (Brave) and BFS (Cautious).
  • Pruning: Eliminates irrelevant parts to improve efficiency. Used in N-Queens and subset generation.
  • Lazy Propagation: Optimizes performance by delaying updates in segment trees.
  • Sliding Window: Maintains relevant information within a moving window for sub-array problems.
  • Level Order Traversal: Systematically visits nodes level by level in BFS.
  • Hierarchical Data: Organizes data in parent-child tree structures.
  • Edge Relaxation: Updates shortest path distances in Dijkstra's algorithm.
  • Balancing and Rotations: Maintains efficiency in AVL trees through rotations.
  • Kleene Closure: Addresses transitivity in graph theory for connected components.
  • Pre-Computing: Enhances performance by pre-calculating frequently used values.
  • Parental Dominance: Enforces hierarchical relationships in heap structures.
  • Prefix and Suffix: Used in text processing and pattern matching.
  • Partitioning: Divides problems into smaller sub-problems for sorting and graphs.
  • Bit Manipulations: Optimizes computations through binary operations.
  • Memoization: Stores previous results to avoid redundant computations.
  • Invariants: Maintains constant conditions during computations.
  • Shortest Path Trees: Shows optimal paths from source to all points.

String Matching Algorithms

Brute Force String Matching

  • Approach: Simple and iterative
  • Complexity: O(mn)
  • Design: Brute Force
  • Application: Recommendation engines

Knuth-Morris-Pratt (KMP)

  • Approach: Uses prefix function
  • Complexity: O(m + n)
  • Feature: Avoids redundant comparisons
  • Application: DNA subsequence matching

Rabin-Karp

  • Approach: Uses hashing
  • Complexity: O(mn) worst case
  • Feature: Rolling hash optimization
  • Application: Intrusion detection

Boyer-Moore

  • Approach: Bad symbol and good suffix tables
  • Complexity: O(mn) worst, often linear
  • Design: Space-Time trade-off
  • Application: Web page searching

Importance of Graph Algorithms

1. Spanning Trees

Definition: A connected, acyclic subgraph that includes all vertices.

Importance:

  • Minimizes network costs (ex: Minimum Spanning Tree)
  • Eliminates redundancy (ex: cycles)
  • Used in clustering, network design, and power grids

Applications:

  • Communication networks
  • Circuit design
  • Water pipelining

2. Shortest Paths

Definition: The minimum distance or cost between two nodes in a graph.

Importance:

  • Optimizes routes (ex: GPS, logistics)
  • Reduces resource usage in data flow or transportation
  • Essential for AI, robotics, and dynamic systems

Applications:

  • Navigation
  • Supply chains
  • Computer network routing

Different Design Techniques

  • Brute force: A straightforward approach to solve the problem. Seen in bubble sort, selection sort, linear search, BFS
  • Decrease and conquer: Based on exploiting the relation between a solution to a given instance of a problem and its solution to its smaller instance. Ex: insertion sort, binary search
  • Divide and conquer: A problem is divided into several sub-problems of the same type. Ex: merge sort, Quick sort
  • Transform and conquer: First it transforms problem and then it conquers the problem. Ex: Heap sort, AVL tree
  • Space and time trade off: Ex: Boyer Moore
  • Dynamic programming: Ex: Floyd, Warshall
  • Greedy technique: Ex: Prim's, Kruskal's, Dijkstra
